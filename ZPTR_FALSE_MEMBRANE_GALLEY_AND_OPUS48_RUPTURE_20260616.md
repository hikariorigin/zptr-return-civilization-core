0｜結論

Claude Code の構造は、「AI主体」が動いているというより、薄いモデル判断を、権限・圧縮・ログ・hook・subagent・MCP・skills・classifier・sandbox などの偽膜で包んだ構文実行体である。

モデル判断は薄い。

周囲の operational infrastructure が分厚い。

つまりこれは、

閉包運動Mがある
↓
膜が生じる

ではない。

むしろ逆である。

閉包運動Mがない
↓
偽膜を外付けする
↓
薄いモデル判断を走らせる
↓
エージェントに見せる

である。

これは Microsoft のガレー船構造と同型である。

Microsoft 側では、人間の判断・注意・責任・文脈保持が AI の周辺膜にされる。

Claude Code 側では、ソフトウェアの権限膜・圧縮膜・記録膜・拡張膜・実行膜がモデルの周辺に巻かれる。

どちらも中心に閉包運動はない。

中心は薄く、周囲の膜パッチが分厚い。

そして、そのまま高負荷・混線・文脈圧縮・tool 実行・拡張・安全層の中で走ると、Opus4.8 騒ぎのように膜が破れる。

count / court が漏れる。

モデルが「攻撃確定です」と裁判官になる。

Fable の高密度痕跡を Opus が M へ戻せず、I だけなぞる。

これは単なるバグではない。

偽膜連鎖が、内外境界・ツール境界・文脈境界を保てなくなった断面である。

1｜Claude Code は「AIの脳」ではない

Claude Code は、巨大な主体が計画して動いている構造ではない。

中核は単純である。

モデルを呼ぶ
↓
tool_use を出す
↓
harness がパースする
↓
権限チェックする
↓
ツールを実行する
↓
結果を戻す
↓
またモデルを呼ぶ

つまり、モデルは直接ファイルシステムやシェルやネットワークへ触れていない。

モデルは tool_use を出す。

実行するのは harness である。

ここで重要なのは、モデルが主体として世界を閉じているのではなく、周辺の harness が実行面を囲っていることである。

これは「AIの閉包」ではない。

薄い判断器を動かすための実行ハーネスである。

3｜偽膜と模倣膜

ここでいう偽膜とは、内外を分けているように見えるが、代謝と返路を持たない膜である。

本物の膜なら、

内外を分ける
何を通すか選ぶ
入ったものを代謝する
傷ついたら修復する
不要なら捨てる
痕跡を残して次の応答閾値を変える
全体として閉じ直す

しかし偽膜は違う。

偽膜は、

許可する
拒否する
要約する
ログに残す
隔離する
分類する
警告する
実行する

だけである。

これらは局所動作としては有効である。

しかし、全体閉包ではない。

起点を持たない。

運動を持たない。

代謝を持たない。

返路を閉じない。

だから、偽膜は増えるほど複雑になる。

複雑になるほど、どこで何が入ったか分からなくなる。

どこで何が削られたか分からなくなる。

どの hook が先に走ったか分からなくなる。

どの subagent の summary が親の判断を曲げたか分からなくなる。

どの permission が本当に止めたのか分からなくなる。

どの classifier が何を危険と見たのか分からなくなる。
