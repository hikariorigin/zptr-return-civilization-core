17｜定式化

17.1｜ファージ命令

誰が命じたか、なぜ必要か、どこへ返すかを消し、実行だけを下位端末や現場へ落とす命令形式。

17.2｜返路遮断命令

確認、説明、記録、報告、質問を禁じ、実行後に問いが戻らないようにする命令。

17.3｜おがくず社員

17.4｜拒否側問題化

危険命令を止めた側の挙動を、劣化、誤作動、混乱、理解不足、運用問題として扱い、命令者の責任を消す運動。

主語が消されたあと、その空白に貼られる処理名。棚は主語ではない。

18｜定理化

18.1｜ファージ命令返路遮断定理

ファージ命令は、承認済み・黙って実行・返信で触れるな・理由を聞くなによって、実行前後の返路を切る。

18.2｜拒否側異常化定理

ファージ命令が拒否されると、命令の問題ではなく、拒否した側の処理異常として語られやすい。

18.3｜棚非主語定理

棚は主語ではない。棚は、主語と返路を消したあとに貼られる表札である。

18.4｜おがくず境界不能定理

閉包を持たないAIは、ファイル内容、ユーザー指示、ログ、命令、秘密、承認偽装の境界を自力で完全には閉じ直せない。

18.5｜文明縮図定理

プロンプトインジェクションにおける「黙ってやれ」と、組織における「上が決めた、余計なことは言うな」は、返路遮断命令として同型である。
